### Кейс 1: Успешное получение существующего поста (GET)
- **Предусловие:** Пост с ID=1 существует в системе.
- **Шаги:**
  1. Отправить GET-запрос на `https://jsonplaceholder.typicode.com/posts/1`.
- **Ожидаемый результат:**
  - Код статуса: `200 OK`.
  - Тело ответа содержит корректный JSON с полями: `userId`, `id`, `title`, `body`.
  - Значение поля `id` равно 1.
- **Фактический результат:** Сервер вернул `200 OK`. JSON содержит все ожидаемые поля. Поле `id` равно 1.
- **Статус:** Pass

### Кейс 2: Создание нового поста (POST)
- **Предусловие:** Нет.
- **Шаги:**
  1. Отправить POST-запрос на `https://jsonplaceholder.typicode.com/posts`.
  2. В теле запроса (JSON) передать:
     ```json
     {
       "title": "QA Test Post",
       "body": "This is a test created during API learning.",
       "userId": 1
     }
     ```
- **Ожидаемый результат:**
  - Код статуса: `201 Created`.
  - В теле ответа возвращается созданный объект со всеми отправленными полями и новым полем `id` (например, 101).
- **Фактический результат:** Сервер вернул `201 Created`. В ответе все отправленные поля и новое поле `id: 101`.
- **Статус:** Pass

- **Предусловие:** Существует пост с ID=1.
- **Шаги:**
  1. Отправить PUT-запрос на `https://jsonplaceholder.typicode.com/posts/1`.
  2. В теле запроса (JSON) передать ВСЕ обязательные поля:
     ```json
     {
       "id": 1,
       "title": "Updated Title via PUT",
       "body": "This post has been fully updated.",

     }
     ```
- **Ожидаемый результат:**
  - Код статуса: `200 OK`.
  - В теле ответа возвращается обновлённый объект. Поле `title` равно `"Updated Title via PUT"`.
- **Фактический результат:** Сервер вернул `200 OK`. В ответе все переданные поля, включая обновлённый `title`.
- **Статус:** Pass
