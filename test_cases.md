# Тест-кейсы: Ручное тестирование REST API (JSONPlaceholder)

## Объект тестирования: Эндпоинт `/posts`

### Кейс 1: Успешное получение существующего поста (GET)
- **Предусловие:** Пост с ID=1 существует в системе.
- **Шаги:**
  1. Отправить GET-запрос на `https://jsonplaceholder.typicode.com/posts/1`.
- **Ожидаемый результат:**
  - Код статуса: `200 OK`.
  - Тело ответа содержит корректный JSON с полями: `userId`, `id`, `title`, `body`.
  - Значение поля `id` равно 1.
- **Фактический результат:** *[Заполняется после выполнения]*
- **Статус:** Pass / Fail

### Кейс 2: Создание нового поста (POST)
- **Предусловие:** Нет.
- **Шаги:**
  1. Отправить POST-запрос на `https://jsonplaceholder.typicode.com/posts`.
  2. В теле запроса (JSON) передать:
     ```json
     {
       "title": "QA Test Post",
       "body": "This is a test created during API learning.",
       "userId": 1
     }
     ```
- **Ожидаемый результат:**
  - Код статуса: `201 Created`.
  - В теле ответа возвращается созданный объект со всеми отправленными полями и новым полем `id` (например, 101).
- **Фактический результат:** *[Заполняется после выполнения]*
- **Статус:** Pass / Fail

### Кейс 3 (Негативный): Попытка получить несуществующий пост (GET)
- **Шаги:**
  1. Отправить GET-запрос на `https://jsonplaceholder.typicode.com/posts/99999`.
- **Ожидаемый результат:**
  - Код статуса: `404 Not Found` (или `200` с пустым объектом, в зависимости от реализации API).
- **Фактический результат:** *[Заполняется после выполнения]*
- **Статус:** Pass / Fail

*Примечание: Это учебные кейсы для практики. Сервер JSONPlaceholder не сохраняет данные, поэтому POST всегда возвращает успех с симуляцией.*
